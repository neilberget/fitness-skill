# Weekly check-in format

`check-ins.md` (in the storage directory chosen per SKILL.md) is append-only and **reverse chronological** (newest entry directly under the header). Entries are short — this is a quick retrospective, not a journal.

## File header (only on first creation)

```markdown
# Weekly check-ins

_Newest first. Append new entries directly below this header._
```

## Entry format

```markdown
## YYYY-MM-DD — Weekly check-in

**Window:** YYYY-MM-DD → YYYY-MM-DD (X sessions logged)
**Overall feel:** [1–2 sentences — energy, motivation, recovery]

### What worked
- ...

### What to drop or change
- ...

### Goal progress / life context
- ...

### Plan for next ~week
- [What you and the user agreed to focus on or adjust]

### Profile changes made
- [One bullet per profile edit, or "none"]

---
```

The trailing `---` separates entries visually.

## What to capture

Pull from the user's actual answers. Don't pad. If they said "felt good, no changes," the entry is two lines and that's fine — the date/cadence is the load-bearing part.

Always note **what concretely changed in `profile.md`** (or "none") so future sessions can trace why a setting shifted.

## What NOT to do

- Don't fabricate sentiment or progress the user didn't report.
- Don't overwrite or edit prior entries — only append new ones at the top.
- Don't duplicate the workout log here. Reference patterns ("strength was up 2 sessions in a row") rather than re-listing sessions.
